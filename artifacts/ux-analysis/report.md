# UX 分析报告: [Game 067] Stretch Arm Bank Job

> **议题**: 手臂延伸偷取金库内现金
> **类型**: 物理 / 搞笑
> **分析视角**: UX / 交互设计
> **协作队友**: tech-analyst

---

## 1. 核心体验脉搏 (Core Experience Pulse)

### 一句话体验
> "手指一拖→手臂拉长→抓回钞票→在警报响起前收手——每个动作要快、要稳。"

### 三阶段交互节奏 (Core Loop: 延伸 → 抓取 → 撤回)

| 阶段 | 持续时间 | 用户动作 | 核心感受 | 风险 |
|------|---------|---------|---------|------|
| **① 延伸** | 0.5–2s | 从起点拖拽手指/鼠标向金库方向 | 伸展、瞄准 | 碰到障碍物触发警报 |
| **② 抓取** | 0.3s (瞬间判定) | 手指移动到钞票位置松手/点击 | 瞬间满足感 | 抓空/抓偏浪费这次机会 |
| **③ 撤回** | 0.5–1s | 手臂自动缩回 | 紧张期待 | 收回途中碰到感应器 |

**UX 关键洞察**: 三个阶段的无缝过渡和清晰的视觉状态区分，是「3秒无教程上手」的核心。

## 2. 操作方式分析 (Controls Analysis)

### 触屏 (主输入)
- **拖拽起点**: 屏幕底部/角落显示玩家角色/手部图标，拖拽从该点开始
- **拖拽过程**: 手指移动方向决定手臂延伸方向，手指与起点的距离决定手臂长度
- **抓取判定**: 手指松开(pointerup)时，手臂末端位置检测是否与钞票/道具重叠
- **触屏优化**:
  - `touch-action: none` 防止浏览器手势干扰
  - 拖拽起点设一个「安全区」(起始点周围30px内松手=取消动作)
  - 拖拽路径上显示实时虚线轨迹，让玩家感知「手臂路径」

### 鼠标 (次要输入)
- **mousedown** 在起点点开始拖拽
- **mousemove** 控制方向和长度
- **mouseup** 触发抓取
- 鼠标光标在拖拽时改为十字准星样式，增强瞄准感

### 键盘 (辅助输入 — 最少支持)
- **方向键**: 控制手臂延伸方向（上/下/左/右）
- **空格/Enter**: 抓取/释放
- 仅作为 fallback，不要求键盘完成全部难度关卡

### 输入支持矩阵
| 输入方式 | 核心操作 | 替代操作 | 支持等级 |
|---------|---------|---------|---------|
| 触屏 | pointerdown + pointermove + pointerup | — | ★★★★★ (主) |
| 鼠标 | mousedown + mousemove + mouseup | — | ★★★★☆ (次) |
| 键盘 | 方向键 + 空格 | — | ★★★☆☆ (fallback) |

## 3. 视觉反馈系统 (Visual Feedback)

### 3.1 手臂状态视觉语言

| 状态 | 视觉表现 | 说明 |
|------|---------|------|
| **空闲 (Idle)** | 手臂蜷缩在起点，微弹跳动画 | 吸引玩家注意 |
| **延伸中 (Extending)** | 手臂从起点沿拖拽方向线性伸长，末端带「抓取圈」 | 抓取圈 = 半透明圆形，让玩家知道「手伸到哪儿了」 |
| **接近钞票** | 抓取圈变为金色/发光 | 距离判定范围内给出正反馈 |
| **抓取成功** | 手臂短暂停顿→收紧→变亮→开始缩回 | 短促的「拉紧」动画 |
| **缩回中 (Retracting)** | 手臂带着钞票匀速缩回，钞票在末端随动摇摆 | 轻微弹性动画增加趣味 |
| **危险接近警报区域** | 手臂变红/脉动 | 距离障碍物<20px时预警 |

### 3.2 金库与场景反馈

| 元素 | 正常状态 | 触发状态 |
|------|---------|---------|
| 钞票 | 静止或微光闪烁 | 被抓取时短暂高亮→跟随手臂 |
| 警报感应器 | 蓝色/绿色发光 | 被触碰→红色闪烁+脉冲扩散 |
| 金库门 | 关闭状态 | 钞票被取出后微微晃动 |
| 激光网格 | 淡红色半透明线 | 被手臂穿过→闪烁+音效警告 |

### 3.3 倒计时 / 警报指示器
- **屏幕边缘光环**: 屏幕四周半透明红色脉冲光环，随剩余时间或警报等级加强
- **进度条**: 游戏顶部固定的红色条，从右向左收缩——玩家可余光感知
- **触发警报时**: 屏幕闪烁+红色脉冲波+震动+音效，强反馈打断当前动作

## 4. 音效与震动反馈 (Audio & Haptic)

| 事件 | 音效 | 震动模式 (ms) |
|------|------|--------------|
| 开始拖拽 | 轻弹簧声 (short pluck) | 15 |
| 手臂延伸 | 持续轻微拉伸音 (音量随长度增大) | — |
| 接近障碍物 | 警告滴声 (快速重复) | 20, 10, 20 |
| 触碰警报 | 刺耳警报声 + 低频嗡鸣 | 100, 50, 100, 50 |
| 抓取钞票 | 满足的「叮」声 | 25 |
| 成功缩回 | 收银机「叮咚」+ 金币声 | 30, 20 |
| 失败/被抓 | 低沉的坠落音 | 80 |
| 通关结算 | 胜利小号旋律 (4音) | 60, 30, 60, 30 |

## 5. 结算与再玩循环 (Settlement & Replay)

### 结算页设计
- **成功**: 绿色主题 → 显示偷取金额/得分 → "继续挑战更高分" 按钮
- **失败**: 红色主题 → 显示触发的警报/扣分原因 → "再来一局" 按钮
- **两种结算页共同要素**:
  - 得分显示 (本局分数 + 本地最高分)
  - 「再来一局」主按钮 (focus 自动落在它上面)
  - 返回 Arcade 按钮
  - 键盘 Enter/Space 也能触发重开

### 本地存储
```
localStorage key: 'stretch_arm_bank_best'
{
  bestScore: 0,        // 最高分
  highestCash: 0,      // 单次偷取最高金额
  gamesPlayed: 0,      // 游玩次数
  alertsTriggered: 0   // 累计触发警报次数 (记忆点)
}
```

## 6. 无障碍与易用性 (Accessibility)

| 需求 | 实现方式 |
|------|---------|
| 色觉障碍 | 不要仅靠颜色区分状态——形状+图标辅助（警报=感叹号，钞票=$符号） |
| 屏幕阅读器 | 关键状态用 `aria-live="polite"` 播报（"手臂延伸中""抓取到100元"） |
| 触屏大/小屏适配 | 拖拽起点区 ≥44px，抓取圈 ≥48px |
| 减少意外操作 | 拖拽起点固定区域+取消拖拽安全区 |
| 键盘导航 | 结算页按钮可 Tab 导航，Enter 触发 |

## 7. 关卡成长与道具 (Progression)

### 初始体验 (前3局)
- 金库布局简单：1排钞票 + 1个缓慢摆动的感应器
- 玩家可以轻松抓取，建立「拖拽→抓取→撤回」的心智模型
- 目标金额低 (如 500元)，容易达成首次通关

### 进阶难度
| 等级 | 新元素 | 核心挑战 |
|------|--------|---------|
| 2 | 增加2个感应器 | 路径规划 |
| 3 | 激光网格 (移动激光线) | 时机掌握 |
| 4 | 风速干扰 (手臂被吹偏) | 补偿操作 |
| 5 | 多重钞票 + 限时金库门 | 优先级判断 |

### 道具系统
| 道具 | 效果 | 获取方式 |
|------|------|---------|
| 手臂加固 | 可碰一次警报不触发 (护盾) | 每局随机掉落 |
| 延长器 | 手臂长度 +50% | 金库角落稀有刷出 |
| 磁铁 | 自动吸附附近钞票 | 连抓3次成功赠送 |
| 减速时钟 | 警报反应延迟2秒 | 连击奖励 |

## 8. 技术实现建议 (从 UX 角度给开发团队)

### 与 tech-analyst 讨论共识

| 讨论议题 | UX 立场 | 技术方案共识 | 理由 |
|---------|---------|-------------|------|
| 渲染方案 | 需要弹性/变形动画 | **Canvas 2D + requestAnimationFrame** | DOM 无法平滑处理手臂形变和实时碰撞 |
| 碰撞检测 | 玩家需要清晰感知边界 | **AABB 碰撞检测** + 视觉警示区域 | 像素级碰撞难以预判，增加挫败感 |
| 警报指示器 | 余光即可感知 | 屏幕边缘脉冲光环 + 进度条 | 让玩家专注拖拽的同时感知威胁 |
| 输入方式 | 触屏优先 | pointerEvents 统一触屏/鼠标 | 三种交互阶段需要清晰状态区分 |

### 建议的项目文件结构

## 10. 风险与注意事项 (Risks)

1. **触屏误操作**: 指尖滑过屏幕容易误触发拖拽 → 起点区域+安全取消区
2. **手臂物理感不足**: 纯直线伸缩太生硬 → 需要轻微的弹性缓动(easing)和摆动惯性
3. **警报太过频繁导致挫败**: 初始关卡感应器要少，让玩家先体验成功
4. **一局时间感知**: 180秒内要保持节奏变化——前60秒简单，60-120秒出现障碍物，120秒+ 警报密集
5. **移动端刘海/底部安全区**: 使用 `safe-area-inset-*` CSS 确保关键UI不被遮挡
