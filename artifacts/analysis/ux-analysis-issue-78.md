# UX 分析报告: Photo Puzzle Restore (Issue #78)

> **游戏名称**: Photo Puzzle Restore  
> **类型**: 拼图/照片  
> **核心玩法**: 拖块 → 还原  
> **操作方式**: 拖拽  
> **工期**: ≤ 3分钟/局  
> **最后更新**: 2026-07-11

---

## 1. 玩家流 (Player Flow)

```
首屏标题 → 选择照片 → 打乱动画 → 拖拽拼块 → 还原完成/时间归零 → 结算页 → 再来一局/换照片
```

### 1.1 首屏 (Title Screen)
参照已有游戏模式 (`games/080-guinea-pig-jigsaw/index.html`), 首屏应包括:
- **游戏标题** + 简短描述 ("拖拽还原被拆散的照片")
- **照片选择区** —— 显示可用的照片包缩略图(网格排列, 可横向滚动)
- **难度选择** —— 3×3 (简单) / 4×4 (困难) 两档
- **最高分/最佳记录** 显示 (从 localStorage 读取)
- **开始按钮** —— 点击后进入游戏

> **验收标准 #1**: 首屏 3 秒内可进入游玩, 无需教程 → UI 布局清晰, 开始按钮显眼, 跳过规则说明

### 1.2 游戏屏 (Game Screen)
- **顶部 HUD**: 倒计时 3:00 (警告 <30s), 当前进度 (已正确放置/N 块), 音效切换按钮
- **拼图区**: 居中正方形网格, 照片拼块随机打乱排列
- **参考缩略图**: 小尺寸还原图展示在角落, 帮助玩家对照
- **底部工具栏**: 提示按钮 (短暂高亮某块正确位置), 重新洗牌按钮 (放弃进度重新打乱)

### 1.3 结算屏 (Result Screen)
- **胜利**: 🎉 动画粒子效果, 显示"完成!", 用时, 得分, 星级评价 (1-3 星)
- **失败(时间归零)**: 显示"时间到!", 已还原块数/总数, 鼓励再来一局
- **两个核心按钮**: "再来一局" (同照片重玩) + "换照片" (回照片选择)
- **新纪录标签** (若本次分数超过 localStorage 历史记录)

> **验收标准 #4**: 失败/胜利结算页有清晰的"再来一局"按钮

## 2. 交互设计 —— 拖拽机制

### 2.1 触控 (Touch) 交互
- **touchstart**: 选中拼块, 轻微放大 + 阴影效果, 视觉反馈
- **touchmove**: 拼块跟随手指移动, 半透明预览显示在目标位置
- **touchend**: 松手时自动吸附到最近的网格位 (snap-to-grid)
  - 吸附阈值: 距正确位置 ≤ 30px 自动吸附对齐
  - 正确拼块: 绿色高亮 + 短音效 + 震动反馈 (10ms)
  - 错误位置: 拼块弹回原位 + 轻微抖动动画

### 2.2 鼠标 (Mouse) 交互
- **pointerdown**: 左键点击选中 (与触控共用 pointer 事件)
- **pointermove**: 拖拽跟随
- **pointerup**: 释放并吸附检测

### 2.3 键盘 (Keyboard) 交互
- **Tab**: 在拼块间轮转焦点
- **方向键 (↑↓←→)**: 移动选中拼块到相邻网格位
- **Space/Enter**: 确认交换选中拼块与目标位拼块
- **Escape**: 取消当前选中

> **验收标准 #3**: 触屏/鼠标/键盘三种输入至少支持两种 → 默认支持触屏 + 鼠标 (pointer events 兼容), 键盘为加分项

### 2.4 拖拽体验关键细节
| 场景 | 反馈 |
|------|------|
| 选中拼块 | 缩放 1.05x + drop-shadow, 轻微上浮动画 |
| 拖拽中 | 拼块跟随指/鼠, 原网格位显示虚线轮廓表示"空" |
| 接近正确位置 (≤20px) | 正确位高亮绿色边框, 暗示意图 |
| 松手在正确位 | 吸附动画 + 成功音效 + 震动 |
| 松手在错误位 | 弹回动画 + 错误音效 + 短震动 |
| 所有块归位 | 胜利动画 + 庆祝音效 |

## 3. 网格大小与难度权衡

| 维度 | 3×3 (9块) | 4×4 (16块) |
|------|-----------|------------|
| 平均完成时间 | 30-90s | 1.5-3min |
| 单块最小尺寸 (340px板) | ~113px ✅ 极易触控 | ~85px ⚠️ 需较精确 |
| 触控误触风险 | 极低 | 中等 |
| 适合玩家 | 新手/儿童 | 有经验/挑战者 |
| 每块含信息量 | 大块内容易识别 | 小块需更仔细观察 |

**建议**: 
- 首屏默认选中 3×3, 4×4 作为高级选项
- 3×3 建议时限 90s, 4×4 建议时限 180s
- 触控目标下限 70px (4×4 在 340px 面板中是 85px, 达标)

## 4. 反馈系统

### 4.1 视觉反馈
- **拼块状态**: 默认 (平整) → 选中 (上浮+阴影) → 正确放置 (绿色边框锁定) → 错误松手 (抖动动画)
- **进度条**: 已正确放置块数 / 总数, 进度条平滑动画
- **倒计时**: 常规白色 → <30s 变为红色 + 脉冲动画
- **胜利动画**: 金色粒子效果 (参照 `080-guinea-pig-jigsaw` 的 star-particle 实现)
- **参考图**: 小型缩略图, 可点击放大查看

### 4.2 音频反馈 (Web Audio API)
参照 `games/096-word-ladder-climb/app.js` 的 tone/sfx 模式:
| 事件 | 音效 | 参数参考 |
|------|------|---------|
| 选中拼块 | 短 click (360Hz, 25ms, square) | `sfxKey()` |
| 正确放置 | 双音确认 (520→780Hz) | `sfxValid()` |
| 错误松手 | 低沉 buzz (160Hz, 140ms) | `sfxInvalid()` |
| 胜利 | 上行和弦 (523→659→784→1046Hz) | `sfxWin()` |
| 失败 | 下行和弦 (330→247→165Hz) | `sfxFail()` |
| 倒计时警告(最后10s) | 每秒 tick, 频率逐秒升高 |

### 4.3 震动反馈 (Haptic)
| 事件 | 模式 (ms) |
|------|-----------|
| 选中拼块 | 10 |
| 正确放置 | 18 |
| 错误松手 | [35, 25, 35] |
| 胜利 | [60, 35, 90] |
| 失败 | [120, 50, 120] |

> **验收标准 #5**: 关键音效与震动反馈齐全

## 5. 计时与得分

### 5.1 计时规则
- 总时限: 180s (3分钟)
- 倒计时显示: `MM:SS` 格式
- 最后 10s: 秒级视觉脉冲 + 音频 tick
- 时间归零 → 立即结算为失败

### 5.2 得分公式
```
基础分 = 正确拼块数 × 100
时间奖励 = 剩余秒数 × 5
星级判定:
  ⭐ (1星): 完成拼图
  ⭐⭐ (2星): 用时 < 2/3 总时限
  ⭐⭐⭐ (3星): 用时 < 1/3 总时限
失败: 无得分 (显示已还原 N/总块数)
```

### 5.3 本地存储
- 存储 key: `photo_puzzle_restore_best`
- 存储字段: `{ bestScore, fastestTime, gamesPlayed, wins, bestGridSize }`
- 结算时自动更新

> **验收标准 #6**: 通关或失败时记录最高分到本地存储

## 6. 进阶与成长: 照片包系统

### 6.1 照片分类
按主题分"照片包", 每个包包含 4-6 张照片:
- **默认包**: 自然风景 (山/海/森林) — 免费解锁
- **更多包**: 城市地标 / 动物 / 美食 / 太空 — 通过完成前一包解锁

### 6.2 解锁规则
- 完成上一包中任意 3 张照片 → 解锁下一包
- 首包(风景)默认解锁
- 每个照片包显示解锁进度: 🔒🔒🔓🔓🔓

### 6.3 首屏照片选择 UI
- 横向滚动的照片卡片列表
- 每张卡片: 缩略图 + 照片名 + 该照片历史最佳成绩
- 已解锁: 正常显示; 未解锁: 灰色锁定图标
- 选中后自动进入游戏

## 7. 无障碍 (Accessibility)

### 7.1 触控目标
- 最小拼块尺寸: 70px (物理像素), 确保手指不会误触相邻块
- 按钮最小: 44×44px (符合 WCAG 2.1 指南)
- 拖拽区: `touch-action: none` 防止浏览器滚动/缩放干扰

### 7.2 色彩对比
- 所有文本对比度 ≥ 4.5:1
- 拼块边框颜色与照片内容确保可区分
- 正确/错误反馈不依赖颜色: 同时使用动画+颜色+音效

### 7.3 动画控制
- 使用 `prefers-reduced-motion` media query 禁用非必要动画
- 关键动画 (正确放置) 保留, 但缩短时长至 150ms

### 7.4 屏幕阅读器
- 拼块使用 `role="button"` 和 `aria-label="拼块 3 行 2 列"`
- 拖拽状态变化通过 `aria-live` 播报
- 结算信息使用 `role="alert"` 自动播报

## 10. 风险与建议

### 高风险
1. **照片素材版权**: 建议使用 CC0 图库 (Unsplash/Pexels) 生成内置照片, 避免版权风险
2. **Canvas 渲染性能**: 4×4 模式下每块需实时裁剪渲染, 低端设备可能卡顿 → 考虑预生成拼块为独立 ImageBitmap

### 低风险
3. **拖拽手感调优**: 吸附阈值和弹回动画需原型测试调整
4. **照片包内容**: 初期 6 张照片足够, MVP 不追求大量内容
5. **触屏拖动 vs 页面滚动**: `touch-action: none` 在游戏区域禁止滚动, 确保不干扰游戏操作
